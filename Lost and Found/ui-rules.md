# 失物招领 APP — UI 设计规范

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **简洁高效** | 降低认知负荷，每页聚焦一个核心任务，减少非必要元素 |
| **一致性** | 相同功能的交互方式、视觉样式保持一致，降低用户学习成本 |
| **反馈及时** | 所有操作（提交、加载、错误）均需即时反馈，杜绝"假死"状态 |
| **容错友好** | 表单提供实时校验，关键操作（删除、标记已找到）需二次确认 |
| **校园风格** | 色调清新明快，贴合校园场景，图标语义清晰 |

## 二、色彩规范

| 用途 | 色值 | 说明 |
|------|------|------|
| **主题色** | `#4A90D9` | 蓝色，用于导航栏、按钮、选中态、链接 |
| **失物标签** | `#E74C3C` | 红色，用于失物类型标签及状态标识 |
| **招领标签** | `#27AE60` | 绿色，用于招领类型标签及状态标识 |
| **主背景** | `#F5F6FA` | 浅灰背景，用于页面底色 |
| **卡片背景** | `#FFFFFF` | 白色，用于信息卡片 |
| **主文字** | `#333333` | 深灰，用于标题及重要文字 |
| **辅助文字** | `#999999` | 浅灰，用于时间、地点等描述性文字 |
| **分割线** | `#EEEEEE` | 极浅灰，用于列表分隔 |
| **危险操作** | `#E74C3C` | 红色，用于删除、退出等按钮 |

## 三、字体规范

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| H1 | 20px | Bold | 页面标题（详情页物品名称） |
| H2 | 18px | Medium | 卡片标题、区块标题 |
| 正文 | 16px | Regular | 表单标签、列表正文 |
| 辅助 | 14px | Regular | 时间、地点、状态标签 |
| 小字 | 12px | Regular | 提示文字、输入框占位符 |

---

## 四、间距规范

| 层级 | 尺寸 | 用途 |
|------|------|------|
| 页面边距 | 16px | 页面左右留白 |
| 卡片内边距 | 14px | 卡片内容与边框间距 |
| 卡片间距 | 12px | 列表卡片之间的纵向间距 |
| 组件间距 | 10px | 相邻表单项之间的间距 |
| 图标间距 | 8px | 图标与文字之间的间距 |
| 紧凑间距 | 6px | 标签、小元素之间 |

## 五、组件规范

### 5.1 信息卡片

```
┌──────────────────────────────┐
│ ┌────┐                       │  ← 卡片内边距 14px
│ │图片│  物品名称（H2）        │
│ │区域│  失物/招领 标签        │  ← 标签：失物红/招领绿
│ │    │  地点 · 时间（辅助色） │
│ └────┘  状态标签              │
└──────────────────────────────┘
```

- 卡片圆角：8px
- 卡片阴影：`0 2px 8px rgba(0,0,0,0.06)`
- 图片区域：宽 80px，高 80px，圆角 6px，`object-fit: cover`
- 点击卡片跳转详情页
- 状态标签：`active` 蓝色 / `found` 绿色 / `closed` 灰色

### 5.2 按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| **主按钮** | 主题色背景 + 白色文字，圆角 6px，高 44px | 提交、确认等主要操作 |
| **次按钮** | 白色背景 + 主题色边框 + 主题色文字，圆角 6px | 取消、返回等次要操作 |
| **文字按钮** | 无背景无边框，主题色文字 | 操作链接（一键复制、查看详情） |
| **悬浮按钮** | 主题色圆形 56×56px，右下角固定定位 | 首页"+"发布按钮 |
| **危险按钮** | 红色文字/背景 | 删除、退出登录、标记已找到 |

### 5.3 输入框

```
┌──────────────────────────────┐
│  标签文字（16px）              │
│  ┌──────────────────────┐   │
│  │  占位符文字（12px #CCC）│   │  ← 高 44px，边框 #E0E0E0
│  └──────────────────────┘   │
│  校验错误提示（12px 红色）    │
└──────────────────────────────┘
```

- 边框：1px solid `#E0E0E0`，聚焦时变为主题色
- 圆角：6px
- 错误状态：边框变红 + 红色提示文字

### 5.4 图片上传

```
┌──┐  ┌──┐  ┌──┐  ┌──┐
│ +│  │图│  │图│  │图│    ← 每个 80×80px，间距 10px
└──┘  └──┘  └──┘  └──┘
```

- 上传按钮：虚线边框 `#CCCCCC`，"+" 图标居中
- 已选图片：显示缩略图，右上角有删除按钮
- 最多 4 张，达到上限后隐藏上传按钮

### 5.5 空状态

```
┌──────────────────────────────┐
│                              │
│         [空状态图标]          │  ← 灰色线框图标
│                              │
│      暂无相关数据（14px #999） │
│                              │
└──────────────────────────────┘
```

- 图标尺寸：80×80px
- 文字颜色 `#999999`，字号 14px
- 用于：搜索无结果、通知列表为空、我的发布为空

### 5.6 Tab 切换

```
┌────────────┬────────────┐
│  失物 ✓    │   招领      │  ← 选中态：底部蓝色指示条 + 主题色文字
└────────────┴────────────┘
```

- 选中态：文字加粗 + 主题色 + 底部 2px 指示线
- 未选中态：辅助色文字
- 动画：指示线滑动过渡 0.2s

## 六、交互规范

| 场景 | 行为 |
|------|------|
| **页面加载** | 显示骨架屏或 loading 动画，避免空白闪烁 |
| **下拉刷新** | 首页列表、个人中心支持下拉刷新，释放后触发 `onPullDownRefresh` |
| **触底加载** | 首页列表滚动到底部自动加载更多，底部显示"加载中..." |
| **表单提交** | 按钮置灰 + loading 状态，防止重复提交；成功后 toast 提示并返回 |
| **图片上传** | 逐个上传，显示上传进度；上传失败提示重试 |
| **二次确认** | 标记已找到、删除信息、退出登录等操作弹出确认对话框 |
| **联系方式交互** | 长按复制；点击调用系统拨号；脱敏展示（138****5678）|

---

## 七、图标规范

- 统一使用 uni-app 内置 icon 或引入 iconfont 图标库
- TabBar 图标：选中态主题色，未选中态灰色 `#999999`
- 图标尺寸：TabBar 24×24px，页面内 18×18px
