# 失物招领 APP — 项目概览

## 项目目标

打造一款面向校园场景的轻量级失物招领小程序，帮助学生在丢失或捡到物品时能够**快速发布信息、智能匹配、高效找回**。项目以 uni-app 技术栈实现前后端一体化，聚焦"发布 → 浏览 → 匹配 → 联系 → 结束"的核心闭环，在课程设计周期内交付一个**可用、可演示**的 MVP 产品。

核心目标：

- **降低发布门槛**：无论是失主还是拾主，都能在 1 分钟内完成信息发布
- **提升匹配效率**：系统自动检索失物库与招领库，关键词匹配后站内通知双方
- **保护用户隐私**：联系方式仅双方可见，避免信息过度暴露
- **跑通完整闭环**：从发布到找回/归还，全流程可追踪、可完结

---

## 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| **失主** | 丢失物品、急于找回的学生 | 发布快、传播广、第一时间联系上拾主；联系方式隐私受保护 |
| **拾主** | 捡到物品的学生或教职工 | 归还意愿强但怕麻烦；发布门槛低，不被过度打扰 |
| **管理员** | 学生会或后勤部门人员 | 审核异常信息、处理争议；低频使用，重权限管理 |

## 功能边界

### MVP 必做功能（P0）

| 功能模块 | 说明 |
|---------|------|
| 注册 / 登录 | 手机号或学号注册登录，基础身份认证 |
| 发布失物信息 | 物品名称、地点、时间、联系方式、图片（最多 4 张）、详细描述 |
| 发布招领信息 | 同上，系统自动检索失物库进行匹配 |
| 首页信息列表 | 失物 + 招领瀑布流展示，按时间倒序，Tab 切换 |
| 关键词搜索 | 按物品名称关键词搜索，支持地点/类型/时间筛选 |
| 信息详情页 | 图片轮播、物品详情、发布者信息、联系方式一键复制/拨打 |
| 匹配逻辑 | 物品名称 + 地点关键词匹配，匹配后站内通知双方 |
| 标记"已找到" | 发布者可标记信息为已找到/已关闭，完结流程 |

### 体验加分功能（P1）

| 功能模块 | 说明 |
|---------|------|
| 图片上传 | 支持拍照或从相册选择，最多 4 张 |
| 分类筛选 | 按地点和物品类型筛选浏览 |

### 明确不做（MVP 排除）

| 排除项 | 原因 |
|--------|------|
| 手机短信 / 推送通知 | 需第三方资质和付费，MVP 用站内通知替代 |
| 地图选点 / LBS 定位 | 需接入地图 SDK，校园场景文字描述更实用 |
| 智能语义匹配 / AI 识别 | 技术坑深，课程设计时间不足以踩坑 |
| 在线聊天（IM） | 等于再做一套聊天系统，成本远大于价值 |
| 信用评价 / 实名认证 | 需对接教务系统，复杂度高且非核心 |
| 悬赏 / 价值评估 | 与核心业务无关 |
| 后台数据统计面板 | 锦上添花，先跑通用户核心流程 |
| 第三方登录（微信/QQ） | MVP 用手机号注册即可，减少 SDK 接入 |

## 演示亮点

1. **完整闭环演示**：现场演示"发布失物 → 发布招领 → 系统自动匹配 → 站内通知 → 标记已找到"的全流程，直观展示产品核心价值
2. **双角色体验**：在同一设备上切换失主/拾主身份，对比不同视角下的功能差异与数据隔离
3. **即时匹配通知**：发布招领后系统实时检索失物库并返回匹配结果，展示自动化匹配能力
4. **隐私保护设计**：联系方式脱敏展示，仅匹配双方可见，一键复制拨号便捷沟通
5. **搜索与筛选**：关键词搜索 + 多条件筛选（地点/类型/时间），展示良好的信息检索体验
6. **图片上传与轮播**：支持多图上传，详情页图片轮播展示，提升信息丰富度
7. **状态生命周期管理**：信息从"寻找中"到"已找到"的状态流转，体现完整的业务闭环
